## 🚨 If Something Goes Wrong During Demo

### Website Crashes

→ Say: "Interesting - let me pull up a screenshot"
→ Show screenshot from phone
→ Continue talking about features

### Someone Asks Technical Question You Don't Know

→ Say: "Great question! That's one we'll validate with beta users"
→ Or: "Our API is flexible enough to handle that"
→ Don't make up technical details

### You Lose Internet

→ Switch to PRESENTATION.html (doesn't need internet)
→ Or walk through code in IDE
→ Have these backups ready

### You Go Over Time

→ Judges will appreciate you cutting short
→ Keep presentations to < 10 minutes
→ Better to end strong than ramble
